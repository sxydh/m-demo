概述
    PV （ Persistent Volume ）是集群存储资源
    PV 支持多种存储实现：
        HostPath
            HostPath 是将当前节点路径进行映射和共享，所以 Pod 如果被调度了其它节点，则在容器内对文件的修改，会反应到相应节点的路径上。
        NFS
        ...
环境
    创建 HostPath 的 PV
        vim pv_hostpath_demo.yaml
            追加
                apiVersion: v1
                kind: PersistentVolume
                metadata:
                  name: pv-hostpath-demo
                spec:
                  capacity:
                    storage: 30Gi
                  accessModes:
                    - ReadWriteMany
                  hostPath:
                    path: /opt/module/pv
                    type: Directory
        kubectl create -f pv_hostpath_demo.yaml -n default
    创建 NFS 的 PV
        前置条件
            NFS 服务端（ hadoop01 ）
                参考 m-demo/nfs-demo/doc/随记.txt
            NFS 客户端（ hadoop01, hadoop02, hadoop03 ）
                参考 m-demo/nfs-demo/doc/随记.txt
        vim pv_nfs_demo.yaml
            追加
                apiVersion: v1
                kind: PersistentVolume
                metadata:
                  name: pv-nfs-demo
                spec:
                  capacity:
                    storage: 5Gi
                  accessModes:
                    - ReadWriteMany
                  nfs:
                    server: 192.168.233.129
                    path: /opt/module/nfs/data/shared
        kubectl create -f pv_nfs_demo.yaml -n default
